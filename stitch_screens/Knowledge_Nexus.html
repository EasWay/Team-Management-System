Architecture V2:
Event Stream Pipeline

The core objective of the V2 architecture is to decouple the ingestion layer from the processing workers. By introducing a durable event log, we ensure zero data loss during traffic spikes.

Sarah

We are evaluating Kafka and Redpanda as the primary message brokers. Benchmarks suggest Redpanda offers better tail latency for our specific payload sizes.

This necessitates a shift from synchronous HTTP calls to asynchronous event publishing.

Ingestion Layer

The ingestion service will act as a dumb pipe, validating schema and pushing to the topic. No business logic should reside here.

producer.ts
const publishEvent = async (payload: EventPayload) => {
  await producer.connect();
  await producer.send({
    topic: 'user-events',
    messages: [{ value: JSON.stringify(payload) }],
  });
};
